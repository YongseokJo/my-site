# Phase 1 — UI Design Contract

> Visual and interaction contract for the framework migration and design system foundation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (to be initialized during Phase 1 implementation) |
| Preset | default (will run `npx shadcn@latest init` with Astro setup) |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, chip padding |
| md | 16px | Default element spacing, component padding |
| lg | 24px | Card padding, section inner padding |
| xl | 32px | Layout gaps between components |
| 2xl | 48px | Major section breaks, minimum touch target size |
| 3xl | 64px | Page-level vertical spacing (above/below hero, between page sections) |

Exceptions: Navigation link touch targets use 48px minimum height for accessibility on all viewports.

| Role | Value | Usage |
|------|-------|-------|

| Secondary (30%) | `#141b2d` | Cards, nav background, elevated surfaces |

| Border | `#1e293b` | Card borders, dividers, nav border |
| Muted surface | `#1a2236` | Hover states, subtle surface differentiation |

### Light Mode (Clean Academic Look)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Page background -- clean white |
| Secondary (30%) | `#f8fafc` | Cards, nav background, elevated surfaces |
| Accent (10%) | `#b8860b` | See reserved list below (darker gold for contrast on white) |
| Destructive | `#dc2626` | Destructive actions only (not present in Phase 1) |
| Foreground | `#0f172a` | Body text on light backgrounds |
| Muted foreground | `#64748b` | Secondary text, metadata, captions |
| Border | `#e2e8f0` | Card borders, dividers, nav border |
| Muted surface | `#f1f5f9` | Hover states, subtle surface differentiation |

### Accent Reserved For (both modes)

- Dark mode toggle icon active state
- Navigation active page indicator (bottom border or highlight)
- Primary CTA button backgrounds (future phases)
- Link hover underline color
- Footer social icon hover state

Accent is NOT applied to: body text, card backgrounds, borders, or navigation text.

## Component Inventory (Phase 1 Only)

These are the components needed for the base layout shell that Phase 1 delivers:

| Component | Source | Purpose |
|-----------|--------|---------|
| BaseLayout | Custom Astro | `<html>`, `<head>`, `<body>` wrapper with dark mode class |
| Navigation | Custom Astro | Top bar horizontal nav with brand + 6 menu items |
| Footer | Custom Astro | Copyright line + social icon links |
| DarkModeToggle | React island (shadcn Button + Lucide icons) | Sun/Moon toggle, persists to localStorage |
| ThemeScript | Inline `<script>` | Runs before paint to read localStorage/system preference, prevents flash |

### Dark Mode Interaction Contract

1. On first visit: read `prefers-color-scheme` media query. Apply matching theme.
2. On toggle click: switch theme class on `<html>` element. Write preference to `localStorage` key `theme`.
3. On subsequent visits: read `localStorage.theme` first. If present, use it. If absent, fall back to system preference.
4. The inline ThemeScript MUST run before any paint (placed in `<head>`) to prevent flash of wrong theme.
5. Toggle icon: Sun icon when dark mode is active (clicking switches to light). Moon icon when light mode is active (clicking switches to dark).

### Navigation Interaction Contract

1. Desktop (>= 768px): Horizontal bar with brand left, nav links right, dark mode toggle far right.
2. Mobile (< 768px): Brand left, hamburger icon right. Tapping hamburger opens a full-width dropdown below the nav bar with vertical link list. Dark mode toggle appears within the mobile menu.
3. Active page: Current page nav link receives accent-colored bottom border (2px).
4. Hover: Nav links show muted surface background on hover with 150ms transition.
5. All interactive nav elements (links, hamburger, toggle) must meet the 48px minimum touch target height.

### Footer Interaction Contract

1. Social icons: Google Scholar, ORCID, GitHub, LinkedIn. Each is a 24x24 SVG icon.
2. Each footer social icon must include an `aria-label` matching the service name (e.g., `aria-label="Google Scholar"`, `aria-label="ORCID"`, `aria-label="GitHub"`, `aria-label="LinkedIn"`).
3. Icon hover: opacity transition from 0.6 to 1.0, 150ms ease.
4. Links open in new tab with `rel="noopener noreferrer"`.

## Tailwind v4 CSS Theme Variables

Phase 1 must configure these CSS custom properties in the Tailwind v4 `@theme` block (replaces `tailwind.config.js` in v4):

```css
@theme {
  --font-sans: "Inter Variable", "Inter", system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
  --color-background: #0a0e1a;
  --color-foreground: #e2e8f0;
  --color-card: #141b2d;
  --color-card-foreground: #e2e8f0;
  --color-primary: #d4a54a;
  --color-primary-foreground: #0a0e1a;
  --color-secondary: #1a2236;
  --color-secondary-foreground: #e2e8f0;
  --color-muted: #1a2236;
  --color-muted-foreground: #94a3b8;
  --color-accent: #d4a54a;
  --color-accent-foreground: #0a0e1a;
  --color-destructive: #ef4444;
  --color-border: #1e293b;
  --color-ring: #d4a54a;
  --radius-lg: 1rem;
  --radius-md: 0.75rem;
  --radius-sm: 0.5rem;
}
```

```css
.light {
  --color-background: #ffffff;

  --color-card: #f8fafc;
  --color-card-foreground: #0f172a;

  --color-primary-foreground: #ffffff;
  --color-secondary: #f1f5f9;
  --color-secondary-foreground: #0f172a;
  --color-muted: #f1f5f9;

  --color-accent-foreground: #ffffff;

}
```
