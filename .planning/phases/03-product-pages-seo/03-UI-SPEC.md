# Phase 3 — UI Design Contract

> Visual and interaction contract for product pages (Enzo-Abyss simulation landing + docs, Readea arXiv app landing + tutorials + forms), site-wide SEO (meta/OG/schema.org), and Pagefind search. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialized, style: base-nova) |
| Preset | base-nova with neutral base color |
| Component library | Radix UI (via shadcn) |

## Spacing Scale

Inherited from Phase 1 — no changes:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, code snippet inline spacing |
| sm | 8px | Compact element spacing, feature icon-to-text gap |
| md | 16px | Default element spacing, card internal padding on mobile, form field spacing |
| lg | 24px | Card padding (desktop), section inner padding, feature card padding |
| xl | 32px | Layout gaps between sections, card grid gap |
| 2xl | 48px | Major section breaks on pages, vertical spacing between page sections |
| 3xl | 64px | Page-level vertical spacing, hero section vertical padding (desktop) |

Exceptions:
- Readea hero section uses 80px top padding on desktop (5rem) to match homepage CosmicHero pattern.
- Device mockup placeholder images: 280px max-height on mobile, 400px max-height on desktop.
- Code block padding: 16px horizontal, 12px vertical (12px is 3x4, within scale).
- Pagefind search input touch target: 44px minimum height for accessibility.

## Typography

Inherited from Phase 1 — no changes to the scale:

| Role | Size | Weight | Line Height | Phase 3 Usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 (regular) | 1.6 | Feature descriptions, documentation prose, tutorial step text, form labels |
| Label | 14px | 400 (regular) | 1.4 | Metadata labels, code comments, form hints, search result metadata, breadcrumb text |
| Heading | 24px | 700 (bold) | 1.25 | Page titles, section headings ("Features", "Quick Start", "Installation"), tutorial titles |
| Display | 36px | 700 (bold) | 1.15 | Readea hero headline only |

Additional typographic rules for Phase 3:
- Product taglines: 16px, weight 400, line-height 1.6, color `muted-foreground`. Used directly below product name.
- Code snippets inline: 14px, weight 400, monospace font `"JetBrains Mono", "Fira Code", "Consolas", monospace`. Background `muted`, padding 2px 6px, border-radius `radius-sm`.
- Code blocks: 14px, weight 400, monospace font. Background `muted`, padding 16px, border-radius `radius-lg`, border 1px `border` color. Line-height 1.6.
- Feature card title: 16px, weight 700. Feature card description: 14px, weight 400, color `muted-foreground`.
- Maximum prose width remains 700px for all documentation and tutorial content.
- Enzo-Abyss product name: rendered at Heading size (24px bold) at page top — not Display size since D-02 specifies "simple header, no large hero".
- Readea product name: rendered at Display size (36px bold) in the hero section.

## Interaction Contracts

### FeatureGrid (Both Products)

1. Feature cards are static — no click interaction, no hover animation beyond subtle border-color transition (150ms).
2. On hover: border color transitions to `muted-foreground` (subtle highlight).
3. Feature icons are decorative — use `aria-hidden="true"` on Lucide icons.
4. Grid is 2 columns on tablet+ (>= 768px), 1 column on mobile. Gap: 24px.

### ReadeaHero

1. Hero is static — no animations on load (matches CosmicHero pattern).
2. Background gradient renders immediately via CSS (no image dependency).
3. Device mockup placeholder uses CSS background color (`muted` surface) with centered text "Screenshot coming soon" in `muted-foreground` color at Label size.
4. CTA button scrolls to nothing — it links to `/contact` page (per D-12, user manually distributes access).
5. Hero text is always light-on-dark regardless of site theme (same as homepage CosmicHero).

### CodeBlock

1. Code block displays with monospace font, syntax-highlighted if Astro's built-in Shiki is available.
2. Copy button: Lucide Clipboard icon (16x16) in top-right corner of code block, absolute positioned.
3. On click: copy code to clipboard, icon briefly changes to Lucide Check icon for 2 seconds, then reverts.
4. Copy button has `aria-label="Copy code to clipboard"`.
5. Code blocks have horizontal scroll for lines that exceed container width — no line wrapping.

### BetaSignupForm

1. All fields required. Validation on blur and on submit.
2. Name: minimum 1 character.
3. Email: valid email format (HTML5 `type="email"` + pattern check).
4. Validation errors appear directly below the field in destructive color, 14px label size.
5. Submit button disabled while submitting (shows "Signing up..." text).
6. On success: form content replaced with success message (see Copywriting).
7. On error: error message appears below the form. Form remains populated for retry.
8. Form service: researcher recommends Formspree free tier (50 submissions/month, no credit card, REST API, works with React). Alternative: Formspark. Executor should use whichever the planner selects.

### FeedbackForm

1. All fields required. Validation on blur and on submit.
2. Category: must select one option (default: no selection, placeholder "Select a category").
3. Name: minimum 1 character.
4. Email: valid email format.
5. Message: minimum 10 characters.
6. Validation errors appear directly below each field in destructive color, 14px label size.
7. Submit button disabled while submitting (shows "Sending..." text).
8. On success: form content replaced with success message.
9. On error: error message appears below the form. Form remains populated for retry.
10. Same form service as BetaSignupForm.

### Pagefind Search (SearchDialog)

1. Pagefind is built at Astro build time — indexes all static HTML output.
2. SearchToggle click opens SearchDialog as a modal (Dialog component).
3. Keyboard shortcut Cmd+K / Ctrl+K opens the dialog from anywhere on the site.
4. Search input autofocuses when dialog opens.
5. Results appear as user types (Pagefind handles debouncing internally).
6. Each result: page title (clickable link), text snippet with highlighted match terms, page path.
7. Click a result: navigate to that page, dialog closes.
8. Press Escape or click backdrop: dialog closes.
9. If no results: show "No results found for '{query}'" in muted-foreground.
10. Pagefind CSS is overridden to match the site's design tokens — do NOT use Pagefind's default styles.

### TutorialNav

1. Previous/Next links are standard anchor tags — no client-side routing.
2. If on the first tutorial: only "Next" link shown.
3. If on the last tutorial: only "Previous" link shown.
4. Links use accent color text with ChevronLeft/ChevronRight Lucide icons.
5. On hover: underline text with 150ms transition.

### Breadcrumb

1. Renders as a horizontal list with "/" separators.
2. All segments except the last are clickable links.
3. Last segment (current page) is plain text in foreground color.
4. Link segments use muted-foreground color, hover accent color with 150ms transition.
5. Uses `nav` element with `aria-label="Breadcrumb"` and `ol` list with `aria-current="page"` on last item.

### SEO Meta Tags (SEOHead)

1. SEOHead extends BaseHead — replaces it on all pages going forward.
2. Every page must include: `<title>`, `<meta name="description">`, `<meta property="og:title">`, `<meta property="og:description">`, `<meta property="og:type">`, `<meta property="og:url">`, `<meta property="og:image">` (fallback to site default if no page-specific image).
3. Twitter card meta: `<meta name="twitter:card" content="summary">`, matching title and description.
4. Canonical URL: `<link rel="canonical" href="{full URL}">` on every page.
5. Publication pages additionally include schema.org `ScholarlyArticle` JSON-LD in a `<script type="application/ld+json">` tag.

### SchemaOrgPublication

1. Each publication generates a JSON-LD block with `@type: "ScholarlyArticle"`.
2. Fields mapped: `name` (title), `author` (array of `Person` objects), `datePublished` (year), `url` (DOI or arXiv link), `publisher` (journal name if available).
3. JSON-LD is rendered in the `<head>` of the publications page, one block per publication.
4. If a publication lacks a field (e.g., no DOI), omit that property — do not use null or empty string.

## Accessibility Requirements

1. All form fields must have associated `<label>` elements (not just placeholder text).
2. Validation error messages must be associated with their fields via `aria-describedby`.
3. SearchDialog must trap focus while open (Dialog component handles this).
4. SearchDialog must be closeable with Escape key.
5. Search results must be navigable with keyboard (Tab through results, Enter to select).
6. Breadcrumb uses semantic `<nav aria-label="Breadcrumb">` with `<ol>` and `aria-current="page"` on the current page segment.
7. Code block copy button has `aria-label="Copy code to clipboard"`.
8. Device mockup placeholder has descriptive `alt` text.
9. Feature card icons use `aria-hidden="true"` (decorative).
10. Category selector (Select component) must be keyboard-navigable (arrow keys to browse, Enter to select).
11. All external links use `rel="noopener noreferrer"` and open in new tab.
