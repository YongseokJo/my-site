| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Elevation of Privilege | IssueForm.tsx | mitigate | RLS policy "Developers create issues" enforces reporter=auth.uid() and approved+role check; client-side getUser() is UX only |
| T-05-06 | Elevation of Privilege | ProposalForm.tsx | mitigate | RLS policy "Developers submit proposals" enforces submitter=auth.uid() and approved+role check |
| T-05-07 | Tampering | IssueForm.tsx | mitigate | issues table has CHECK constraints on status and priority; invalid values rejected by Postgres |
| T-05-08 | Information Disclosure | ProposalList.tsx | mitigate | RLS policy restricts viewers to approved proposals only; unapproved users see nothing |
| T-05-09 | Tampering | User-submitted content | mitigate | React JSX auto-escapes all rendered strings; never use dangerouslySetInnerHTML for user content |
</threat_model>
