**Filter chips section (per D-05, UI-SPEC):**
- Extract unique roles from publications: map "first-author" to display "First Author", "co-author" to "Co-Author"
- Extract unique years, sorted descending
- Render chips using shadcn Badge component:
  - Active chip: `variant="default"` with `className="cursor-pointer bg-primary text-primary-foreground hover:bg-primary/90"`
  - Inactive chip: `variant="outline"` with `className="cursor-pointer hover:bg-muted"`
- Chips in a horizontal row with `flex gap-2 overflow-x-auto pb-2` and `-webkit-overflow-scrolling: touch` for mobile horizontal scroll (per UI-SPEC)
- Click handler toggles the role/year in the active set
- Filter logic: within same category = OR, across categories = AND
  - If activeRoles is empty AND activeYears is empty: show all
  - If activeRoles has values: publication must match at least one active role
  - If activeYears has values: publication must match at least one active year
  - Both active: must match role AND year (AND across categories)
- Group chips visually: role chips first, then a small visual separator (thin vertical line or extra gap-4), then year chips

- Render filtered publications as shadcn Card components
- Card layout, each card with `className="bg-card border border-border"` (shadcn Card handles this):

    - Authors: 16px regular, with "Jo, Yongseok" highlighted. Implement `highlightAuthor` function:

      Split authors string by /(Jo, Yongseok)/

      Per D-06 and PUBL-03.
    - Venue + Year: `<p className="text-sm text-muted-foreground mt-1">{pub.venue}, {pub.year}</p>`

      - For each non-empty link (arxiv, doi, ads): render a shadcn Button with `variant="outline"` and `size="sm"`:
        - arXiv button: text "arXiv", href = pub.links.arxiv
        - DOI button: text "DOI", href = pub.links.doi
        - ADS button: text "ADS", href = pub.links.ads
