**Validation (per UI-SPEC interaction contract):**
```typescript
function validate(field: keyof FormData, value: string): string | undefined {
  if (field === "name" && !value.trim()) return "Name is required.";
  if (field === "email" && !/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value)) return "Please enter a valid email address.";
  if (field === "message" && value.trim().length < 10) return "Message must be at least 10 characters.";
  return undefined;
}
```
- Validate on blur (onBlur handler on each field) and on submit
- Error messages appear below the field: `<p className="text-sm text-destructive mt-1">{error}</p>`
- Use `aria-describedby` linking error message to field

**Submit handler (per research Pattern 3 — Netlify Forms with React island):**
```typescript
async function handleSubmit(e: React.FormEvent) {
  e.preventDefault();
  // Validate all fields
  const newErrors: FormErrors = {};
  for (const field of ["name", "email", "message"] as const) {
    const err = validate(field, formData[field]);
    if (err) newErrors[field] = err;
  }
  if (Object.keys(newErrors).length > 0) {
    setErrors(newErrors);
    return;
  }
  setStatus("submitting");
  try {
    const body = new URLSearchParams({
      "form-name": "contact",
      name: formData.name,
      email: formData.email,
      message: formData.message,
    });
    const response = await fetch("/", {
      method: "POST",
      headers: { "Content-Type": "application/x-www-form-urlencoded" },
      body: body.toString(),
    });
    if (!response.ok) throw new Error("Submit failed");
    setStatus("success");
  } catch {
    setStatus("error");
  }
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input -> Netlify Forms | Contact form accepts user input (name, email, message) |
| Client -> Server | Form data submitted via POST to Netlify |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | Contact form submission | mitigate | Client-side validation (email format, message length); Netlify built-in spam filter + honeypot field (bot-field). Form data only shown in Netlify dashboard, never rendered on site (no XSS vector). |
| T-02-06 | Denial of Service | Contact form spam | mitigate | Netlify honeypot field (`netlify-honeypot="bot-field"`) + Netlify's built-in Akismet spam filtering. Rate limiting handled by Netlify. |
| T-02-07 | Information Disclosure | CV PDF download | accept | CV is intentionally public — user uploads it for anyone to download. No sensitive content beyond what user chooses to share. |
| T-02-08 | Spoofing | Email field | mitigate | HTML5 type="email" + regex validation on client. Netlify handles server-side. No email is sent FROM the form (Netlify stores submissions). |
</threat_model>
