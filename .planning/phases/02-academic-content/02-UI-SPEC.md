# Phase 2 — UI Design Contract

> Visual and interaction contract for the academic content pages: homepage with cosmic hero, publications with filtering, research projects with expand-in-place cards, about/CV page, and contact form. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialized, style: base-nova) |
| Preset | base-nova with neutral base color |

## Spacing Scale

Inherited from Phase 1 — no changes:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, filter chip icon spacing |
| sm | 8px | Compact element spacing, chip padding, link button gaps |
| md | 16px | Default element spacing, card internal padding on mobile |
| lg | 24px | Card padding (desktop), section inner padding |
| xl | 32px | Layout gaps between components, card grid gap |
| 2xl | 48px | Major section breaks, hero section vertical padding (mobile) |
| 3xl | 64px | Page-level vertical spacing, hero section vertical padding (desktop) |

Exceptions:
- Hero section uses 80px top padding on desktop (5rem) to create immersive entry. This is the only value outside the 8-point scale, justified by the full-bleed hero design.
- Filter chip touch targets: 36px minimum height (chips are dense inline elements, not primary navigation).
- Publication card link buttons (DOI, arXiv, ADS): 32px minimum height with 8px horizontal padding.

## Typography

Inherited from Phase 1 — no changes to the scale:

| Role | Size | Weight | Line Height | Phase 2 Usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 (regular) | 1.6 | Publication authors, project descriptions, bio text, form labels, hero subtitle |
| Label | 14px | 400 (regular) | 1.4 | Filter chips, publication venue/year, metadata, form hints |
| Heading | 24px | 700 (bold) | 1.25 | Page titles (Publications, Research, About, Contact), section headings, publication card titles |
| Display | 36px | 700 (bold) | 1.15 | Hero headline only ("First Star Clusters & Black Holes in the Early Universe") |

Additional typographic rules for Phase 2:
- Hero subtitle: 16px, weight 400, line-height 1.6, color `muted-foreground`. Used for "Computational Astrophysicist & ML Researcher" below the headline.
- Publication title in cards: 16px, weight 700 (bold). Distinguishes publication titles from author lists through both weight and color contrast.
- Author highlighting: "Jo, Yongseok" rendered at weight 700 (bold) within the otherwise weight-400 author string.
- Maximum prose width remains 700px for body content (about page bio, project descriptions).
- Hero headline max-width: 600px centered, preventing overly long lines on wide screens.

### Hero Overlay

The cosmic hero background image must have a color overlay to ensure text readability:
- Dark mode: `linear-gradient(to bottom, rgba(10, 14, 26, 0.7), rgba(10, 14, 26, 0.95))` over the image
- Light mode: `linear-gradient(to bottom, rgba(10, 14, 26, 0.6), rgba(10, 14, 26, 0.9))` — intentionally dark in both modes for the cosmic effect. Hero text always renders in light colors (`#e2e8f0` foreground, `#94a3b8` subtitle) regardless of site theme.

## Interaction Contracts

### CosmicHero

1. Hero is static — no animations on load (performance first, zero JS).
2. Background image loads with `loading="eager"` and `fetchpriority="high"` (above the fold).
3. Image must have a CSS fallback background color matching the dominant dark color (`#0a0e1a`) so the hero area is never blank during image load.
4. Hero text is always light-on-dark regardless of site theme mode.

### FilterChips (Publications)

1. Initial state: no chips active — all publications shown.
2. Click a chip to activate it (accent background). Click again to deactivate.
3. Multiple chips can be active simultaneously.
4. Filter logic: within the same category (e.g., role), chips are OR. Across categories (role + year), chips are AND.
5. Filtered results update immediately — no loading state needed (client-side filter of 11 items).
6. If no publications match active filters: show empty state (see Copywriting).
7. Chips render in a horizontal row. On mobile, if they overflow, allow horizontal scroll with `overflow-x: auto` and `-webkit-overflow-scrolling: touch`. No wrapping.
8. URL does not update with filter state (too few items to warrant deep linking).

### ResearchProjectCard (Expand-in-Place)

1. Default state: collapsed card showing icon, title, brief description, status badge, chevron-down icon.
2. Click anywhere on the card to expand.
3. Expanded state: card grows in height to reveal full description, related publications, and status details. Chevron rotates to chevron-up.
4. Height transition: 200ms ease with `overflow: hidden` during animation.
5. Only one project can be expanded at a time. Expanding a second project collapses the first.
6. On mobile: same behavior, no layout changes.
7. Expanded content padding: 24px, matching the card padding.

### ContactForm

1. All fields required. Validation on blur and on submit.
2. Name: minimum 1 character.
3. Email: valid email format (HTML5 `type="email"` + pattern check).
4. Message: minimum 10 characters.
5. Validation errors appear directly below the field in destructive color, 14px label size.
6. Submit button disabled while submitting (shows "Sending..." text).
7. On success: form content replaced with success message (see Copywriting).
8. On error: error message appears below the form (see Copywriting). Form remains populated so user can retry.
9. No CAPTCHA — Netlify Forms has built-in spam filtering.

### Publication Link Buttons

1. Each button (arXiv, DOI, ADS) opens in a new tab with `rel="noopener noreferrer"`.
2. Only render buttons for links that exist (publications.yaml has empty string for missing links).
3. Button style: outlined variant (border, no fill). On hover: accent border color + accent text color with 150ms transition.
4. Each button has an `aria-label` like "View on arXiv" for screen readers.

### Avatar (About Page)

1. Avatar image uses `loading="lazy"` (below the fold on desktop, may be visible on mobile).
2. Rendered as a 120x120 circle with `border-radius: 50%` and 2px accent border.
3. Fallback: if image fails to load, show a 120x120 circle with muted background and the initials "YJ" centered in foreground color at Heading size.

## Accessibility Requirements

1. All images (hero background, avatar, project thumbnails) must have descriptive `alt` text.
2. Filter chips must be keyboard-navigable (Tab to focus, Enter/Space to toggle).
3. Expanded research project content must be announced to screen readers (use `aria-expanded` on the card trigger).
4. Contact form fields must have associated `<label>` elements (not just placeholder text).
5. Validation error messages must be associated with their fields via `aria-describedby`.
6. External link buttons must have `aria-label` indicating destination (e.g., "View on arXiv").
7. Hero text must maintain WCAG AA contrast ratio (4.5:1) against the overlay — light text on dark overlay achieves this.
8. Color alone must not convey filter state — active chips have both accent background AND a visible border/shape change.
