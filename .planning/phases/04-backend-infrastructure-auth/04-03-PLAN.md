Create `.github/workflows/supabase-keepalive.yml` -- per D-12 (5-day schedule) and D-13 (HTTP to REST API):

CRITICAL per Research Pitfall 5: The ping MUST execute an actual database query (SELECT from a table), not just hit the REST API root. A simple health check may not count as database activity and the project could still be paused.

```yaml
name: Supabase Keep-Alive

on:
  schedule:
    # Every 5 days at 00:00 UTC (days 1,6,11,16,21,26 of each month)
    # Per D-12: prevents 7-day inactivity pause on Supabase free tier
    - cron: '0 0 */5 * *'
  workflow_dispatch: # Allow manual trigger for testing

jobs:
  ping:
    runs-on: ubuntu-latest
    steps:
      - name: Ping Supabase database
        run: |
          # Per D-13 + Research Pitfall 5: Execute actual SELECT query
          # to count as database activity (not just HTTP health check)
          response=$(curl -sf \
            "${{ secrets.SUPABASE_URL }}/rest/v1/profiles?select=id&limit=1" \
            -H "apikey: ${{ secrets.SUPABASE_ANON_KEY }}" \
            -H "Authorization: Bearer ${{ secrets.SUPABASE_ANON_KEY }}")
          echo "Keep-alive ping successful: $response"
```

**Important for the user:** After pushing this file to GitHub, you must add two repository secrets:
- `SUPABASE_URL` -- the Supabase project URL (same value as PUBLIC_SUPABASE_URL in .env)
- `SUPABASE_ANON_KEY` -- the Supabase anon key (same value as PUBLIC_SUPABASE_ANON_KEY in .env)

Set these at: GitHub repo -> Settings -> Secrets and variables -> Actions -> New repository secret
