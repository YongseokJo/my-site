## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Astro 6 removed output: "hybrid"**
- **Found during:** Task 1, Step 2
- **Issue:** Plan specified changing output from "static" to "hybrid", but Astro 6.x removed hybrid mode. The build error stated: "The output: 'hybrid' option has been removed. Use output: 'static' (the default) instead, which now behaves the same way."
- **Fix:** Kept output: "static". In Astro 6, static mode natively supports per-page SSR opt-in via `export const prerender = false`, which is the same behavior hybrid mode provided.
- **Files modified:** astro.config.mjs (no change from original)
- **Commit:** 9a51cc4

**2. [Rule 1 - Bug] Middleware crash during static prerendering**
- **Found during:** Task 1, Step 6
- **Issue:** Middleware attempted to create a Supabase client during build-time static prerendering when PUBLIC_SUPABASE_URL and PUBLIC_SUPABASE_ANON_KEY are not available, causing build failure on all prerendered pages.
- **Fix:** Added early return guard in middleware that checks for env var availability before creating the Supabase client. During prerendering, locals.user is set to null and the middleware passes through.
- **Files modified:** src/middleware.ts
- **Commit:** 9a51cc4
