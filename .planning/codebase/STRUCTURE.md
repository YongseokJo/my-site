## Directory Purposes

**assets/:**
- Purpose: Hugo asset pipeline—CSS files processed, minified, and fingerprinted
- Contains: SCSS/CSS stylesheets
- Key files: `assets/css/main.css` (primary styles)

**content/:**
- Purpose: Source of truth for page content
- Contains: Markdown files with YAML frontmatter metadata
- Key files:
  - `content/_index.md` (homepage)
  - `content/publications/_index.md` (uses shortcode to render publications)
  - `content/projects/_index.md` (project descriptions)
  - `content/about/_index.md` (about page)

**data/:**
- Purpose: Structured data files for dynamic content
- Contains: YAML files with structured records
- Key files:
  - `data/first_pub.yaml` (first-author publications)
  - `data/co_pub.yaml` (co-authored publications)

**layouts/:**
- Purpose: Hugo template system—defines page structure and rendering logic
- Contains: HTML files with Go templating syntax
- Structure:
  - `layouts/index.html`: Custom homepage layout (overrides default)
  - `layouts/_default/baseof.html`: Master template all pages extend
  - `layouts/_default/single.html`: Single-page layout (articles)
  - `layouts/_default/list.html`: List-page layout (sections)
  - `layouts/partials/`: Reusable template components
  - `layouts/shortcodes/`: Custom Markdown shortcodes

**layouts/partials/:**
- Purpose: Modular template fragments included in multiple pages
- Key files:
  - `nav.html`: Header with navigation menu
  - `footer.html`: Page footer
  - `publications.html`: Publications list renderer
  - `emailcopy-script.html`: Client-side email copy functionality

**static/:**
- Purpose: Static assets served directly to public (no processing)
- Contains: Images, icons, downloads, SEO files
- Key files:
  - `static/icons/`: SVG social media icons
  - `static/images/`: Avatar image, backgrounds
  - `static/robots.txt`: SEO configuration

**scripts/:**
- Purpose: Build and automation scripts
- Key files: None visible; directory present for future utilities

**public/:**
- Purpose: Generated HTML output (build artifact, NOT committed)
- Generated by: `hugo` command
- Deployed by: Netlify CI/CD

## Key File Locations

**Entry Points:**
- `layouts/index.html`: Homepage template (renders `content/_index.md`)
- `config.yaml`: Site configuration and build parameters

**Configuration:**
- `config.yaml`: Site metadata, menu structure, base URL, parameters
- `netlify.toml`: Netlify build environment (Hugo version)
- `before_publish.sh`: Pre-build cleanup and optimization script

**Core Logic:**
- `layouts/_default/baseof.html`: HTML document structure, CSS loading, partial injection
- `layouts/index.html`: Homepage hero layout with two-column responsive grid
- `layouts/partials/publications.html`: Publication list rendering with author highlighting
- `assets/css/main.css`: All styling (responsive design, components)

**Testing:**
- Not applicable—static site generator has no automated tests

## Naming Conventions

**Files:**
- Markdown files: lowercase with underscores (`_index.md`, `first_pub.yaml`)
- Template files: lowercase with underscores, `.html` extension
- CSS files: lowercase with underscores
- Data files: lowercase with underscores, `.yaml` extension
- Directories: lowercase with underscores (`layouts/_default`, `assets/css`)

**Directories:**
- Hugo special directories: lowercase, single names (`layouts`, `content`, `assets`, `data`, `static`)
- Subdirectories: lowercase, underscore-prefixed for Hugo internals (`_default`, `_index.md`)
- User content sections: lowercase, hyphenated (`publications`, `projects`)

**CSS Classes:**
- Pattern: kebab-case (hyphenated)
- Examples: `.hero-inner`, `.card-header`, `.pub-title`, `.social-circle`
- BEM-like structure: `.component-element` or `.component-element--modifier`

**Hugo Templates:**
- Pattern: Curly brace syntax for Go templating
- Variables: `.Title`, `.Content`, `.Site.Params`, `.Pages`
- Functions: `partial`, `block`, `range`, `with`, `define`

## Where to Add New Code

**New Page:**
1. Create Markdown file in `content/section/_index.md` with frontmatter (title, description)
2. Add entry to `config.yaml` menu structure (if top-level page)
3. Override layout if needed by creating `layouts/section/single.html` or use default `layouts/_default/single.html`

**New Content Section:**
1. Create directory under `content/` (e.g., `content/blog/`)
2. Create `_index.md` in section directory (list page)
3. Add child `*.md` files for individual items
4. Hugo will automatically render with `layouts/_default/list.html` and `layouts/_default/single.html`

**New Reusable Template Component:**
1. Create partial in `layouts/partials/component-name.html`
2. Include in templates with `{{ partial "component-name.html" . }}`
3. Pass data via context (`.Page`, `.Site`, custom parameters)

**New Custom Shortcode:**
1. Create file in `layouts/shortcodes/shortcode-name.html`
2. Use in Markdown content as `{{< shortcode-name >}}` or `{{< shortcode-name param="value" >}}`
3. Access parameters via `.Get "param"`

**New Data Source:**
1. Create YAML file in `data/` directory (e.g., `data/awards.yaml`)
2. Access in templates via `site.Data.awards`
3. Iterate with `{{ range .Site.Data.awards }}...{{ end }}`

**New Styling:**
1. Add CSS to `assets/css/main.css` or create new file and import it
2. Follow naming convention: `.component-name` with kebab-case
3. Use CSS variables from `:root` for colors and sizing
4. Include media queries for responsive breakpoints (768px is primary)

## Special Directories

**public/:**
- Purpose: Build output directory
- Generated: Yes (regenerated on each `hugo` build)
- Committed: No (listed in `.gitignore`)
- Contains: Complete static website ready for deployment

**resources/_gen/:**
- Purpose: Hugo's asset pipeline output (fingerprinted CSS, etc.)
- Generated: Yes (cleaned by `before_publish.sh`)
- Committed: No
- Contains: Minified and fingerprinted versions of `assets/`

**.git/:**
- Purpose: Git repository metadata
- Committed: Yes (always present)
- Contains: Repository history and configuration
